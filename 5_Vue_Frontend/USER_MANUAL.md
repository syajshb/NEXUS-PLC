## 1. 系统概述

### 1.1 系统简介

**工业数据监控系统 (IMS Monitor)** 是一款面向制造业的全栈工业物联网平台，集成设备状态实时监控、生产数据采集分析、能耗智能管理、报警事件处理及控制指令下发等核心功能模块。

### 1.2 核心特性

| 特性 | 描述 |
|------|------|
| 实时监控 | 设备状态实时监控，毫秒级数据更新 |
| 数据分析 | 生产数据采集分析，智能报表生成 |
| 能耗管理 | 能耗智能管理，节能减排优化 |
| 报警处理 | 报警事件实时推送，快速响应处理 |
| 远程控制 | 控制指令下发，远程设备控制 |
| 数据持久化 | 历史数据存储，支持数据分析追溯 |

### 1.3 系统架构

```
┌─────────────────────────────────────────────────────────────────┐
│                        前端 (Vue 3)                            │
│  ┌───────────────────────────────────────────────────────────┐  │
│  │ 大屏监控 │ 设备状态 │ 生产数据 │ 能耗数据 │ 报警记录 │ 统计分析 │  │
│  └───────────────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────────────┘
                              │ WebSocket / HTTP
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│                        后端 (Java Spring Boot)                  │
│  ┌───────────────────────────────────────────────────────────┐  │
│  │  REST API │ WebSocket服务 │ 业务逻辑层 │ 数据访问层        │  │
│  └───────────────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────────────┘
                              │ JDBC
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│                        数据库 (MySQL)                           │
│  ┌───────────────────────────────────────────────────────────┐  │
│  │ 设备表 │ 状态表 │ 生产数据表 │ 能耗表 │ 报警表 │ 指令表    │  │
```

## 2. 快速开始

### 2.1 环境要求

| 环境 | 版本要求 |
|------|----------|
| Node.js | >= 18.0.0 |
| NPM | >= 9.0.0 |
| Java | >= 17.0.0 |
| MySQL | >= 8.0.0 |

### 2.2 启动步骤

#### 前端启动
```bash
# 进入前端目录
cd d:\RealPractice\back\5_Vue_Frontend

# 安装依赖
npm install

# 启动开发服务器
npm run dev

# 构建生产版本
npm run build
```


## 3. 功能模块

### 3.1 大屏监控

**功能描述**: 实时展示设备运行状态、生产统计、能耗数据等核心指标的可视化大屏。

**主要功能**:
- 设备总数、运行中、报警数量统计
- 实时趋势图表（温度、湿度、压力、功率）
- 设备状态分布饼图
- 最新报警列表
- 生产统计（产量、良品率、效率、运行时长）
- 能耗监控（用电、用水、用气）
- 设备列表（支持搜索过滤）

### 3.2 设备状态监控

**功能描述**: 设备状态的上报与查询管理。

**主要功能**:
- 设备状态上报（温度、湿度、压力、耗电量、运行时长）
- 设备状态查询（支持按设备编码过滤）
- 设备状态历史记录

### 3.3 生产数据

**功能描述**: 生产数据的采集与分析。

**主要功能**:
- 生产数据上报（产量、良品率、速度、效率）
- 历史数据查询
- 时间范围查询

### 3.4 能耗数据

**功能描述**: 能耗数据的统计与管理。

**主要功能**:
- 能耗数据上报（用电量、用水量、用气量）
- 历史能耗查询
- 时段总能耗统计

### 3.5 报警记录

**功能描述**: 报警事件的管理与处理。

**主要功能**:
- 报警新增
- 报警列表查询
- 报警处理（标记为已处理）
- 获取未处理报警数量

### 3.6 日统计数据

**功能描述**: 每日数据统计报表。

**主要功能**:
- 日统计查询
- 日期范围统计

### 3.7 控制指令

**功能描述**: 设备控制指令的下发与管理。

**主要功能**:
- 创建指令
- 查询待执行指令
- 查询指令详情
- 更新指令状态
- 查询设备指令历史

### 3.8 设备配置

**功能描述**: 设备基础信息的管理。

**主要功能**:
- 设备列表查询
- 添加设备
- 更新设备信息
- 查询设备详情
- 删除设备

## 4. 使用指南

### 4.1 首页导航

1. **进入系统**: 点击"进入系统"按钮进入大屏监控页面
2. **了解更多**: 点击"了解更多"按钮平滑滚动到功能特性区域
3. **导航链接**: 点击顶部导航链接快速跳转到对应区域

### 4.2 大屏监控操作

1. **图表切换**: 使用下拉菜单切换温度、湿度、压力、功率图表
2. **设备搜索**: 在搜索框输入设备编号或名称进行过滤
3. **查看全部报警**: 点击"查看全部"按钮跳转到报警记录页面

### 4.3 服务流程查看

服务流程展示了完整的数据采集链路：
- **PLC**: 可编程逻辑控制器，采集现场设备数据（OPC UA协议）
- **KepServerEX**: 工业数据网关，OPC UA服务器
- **Python**: 数据采集服务，实时数据处理（HTTP/REST协议）
- **MySQL**: 关系型数据库，持久化存储
- **后端**: Java服务层，业务逻辑处理
- **前端**: 可视化展示，实时监控大屏（WebSocket协议）

### 4.4 数据上报操作

1. 选择对应的数据类型标签（如"上报状态"、"上报数据"等）
2. 填写表单字段（带 * 号为必填项）
3. 点击"提交"按钮
4. 查看响应结果（5秒后自动消失）

### 4.5 数据查询操作

1. 选择查询标签
2. 填写查询条件（可选）
3. 点击"查询"按钮
4. 查看表格结果

## 6. 数据采集流程

### 6.1 完整流程

```
PLC 设备
   │
   ▼ OPC UA
KepServerEX
   │
   ▼ OPC UA
Python 采集服务
   │
   ▼ HTTP/REST
Java 后端
   │
   ▼ JDBC
MySQL 数据库
   │
   ▼ WebSocket / HTTP
Vue 前端
```

### 6.2 流程说明

1. **PLC 层**: 现场可编程逻辑控制器，负责采集传感器数据
2. **KepServerEX**: 工业数据网关，作为 OPC UA 服务器接收 PLC 数据
3. **Python 采集服务**: 通过 OPC UA 协议读取 KepServerEX 数据，处理后通过 HTTP 发送到后端
4. **Java 后端**: 接收数据，进行业务处理，存储到数据库
5. **MySQL 数据库**: 持久化存储所有监控数据
6. **Vue 前端**: 通过 WebSocket 实时接收数据更新，展示可视化大屏

### 6.3 协议说明

| 环节 | 协议 | 说明 |
|------|------|------|
| PLC ↔ KepServerEX | OPC UA | 工业标准协议，实时数据传输 |
| KepServerEX ↔ Python | OPC UA | 数据采集协议 |
| Python ↔ 后端 | HTTP/REST | 标准接口协议 |
| 后端 ↔ 数据库 | JDBC | 数据库连接 |
| 前端 ↔ 后端 | WebSocket/HTTP | 实时推送与数据查询 |

## 7. 常见问题

### 7.1 前端无法连接后端

**问题**: 前端显示"服务未连接"

**解决方案**:
1. 确认后端服务已启动
2. 检查后端端口是否为 8080
3. 确认防火墙未阻止端口访问
4. 检查前端配置的 API 地址是否正确

### 7.2 数据上报失败

**问题**: 提交数据后显示"请求失败"

**解决方案**:
1. 检查表单必填字段是否完整
2. 确认设备编码格式正确
3. 检查数值类型是否正确（数字字段不要输入字符串）
4. 确认后端服务正常运行

### 7.3 大屏数据不更新

**问题**: 大屏数据长时间不变化

**解决方案**:
1. 检查 WebSocket 连接状态
2. 确认后端数据采集服务正常
3. 检查 PLC 连接状态
4. 查看浏览器控制台是否有错误信息

### 7.4 报警无法处理

**问题**: 点击"确认处理"后无响应

**解决方案**:
1. 确认报警 ID 输入正确
2. 检查处理人字段是否填写
3. 确认后端服务正常

### 7.5 搜索功能不生效

**问题**: 输入搜索关键词后列表无变化

**解决方案**:
1. 确认输入的关键词正确
2. 检查是否存在匹配的设备
3. 点击搜索按钮或按回车键触发搜索
